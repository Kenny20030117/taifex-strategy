# Project Rules: 台指期外資籌碼策略 (Taiwan Index Futures Foreign Chip Strategy)

本文件定義了「每日策略 (台指期外資籌碼策略)」專案的最高指導原則與核心邏輯。任何對軟體的修改或功能擴充，**必須**首先閱讀並遵守本規範。
若需更動任何規則 (Rule) 或規範 (Specification)，**必須**事先詢問使用者並獲得明確授權。

## 0. AI 行為準則與維護規範 (AI Conduct & Maintenance)
*   **精確執行需求**：使用者的特殊文字設定（如雙空時顯示「偏空看待，逢低買進」），**必須**嚴格遵守，禁止 AI 自行「修正」為逢高賣出等常規邏輯。
*   **穩定優先**：API 爬蟲模組 (`get_taifex_data`, `get_twse_spot`) 已根據爬取目標網站（期交所、證交所）的表格結構與欄位索引進行最佳化，**非必要嚴禁更動解析邏輯**。
*   **防禦性編碼**：修改關鍵邏輯前，必須先確認現有邏輯，並使用 `view_file` 閱讀完整上下文，禁止憑空猜測程式碼行數。
*   **修改限制 (LOCKED)**：
    *   **禁止** 修改定義好的 UI 顏色佈局與高對比（亮白基底）看盤設計。
    *   **禁止** 移除跨網域限制（CORS）的伺服器端抓取架構（必須維持 Python 後端抓取）。

## 1. 開發環境與 API 規範 (Environment & API)
*   **執行環境**：以 Python 作為核心語言，搭配 **Streamlit** 框架作為全端網頁展示。
*   **啟動方式**：透過根目錄下的 `start.bat` 執行 `python -m streamlit run app.py`。
*   **關鍵套件**：
    *   `streamlit`：UI 呈現與互動。
    *   `pandas`：解析 HTML 表格結構。
    *   `requests`、`urllib3`：API 端點請求與 SSL 驗證處理。
*   **爬蟲規範**：
    *   向 TWSE (證交所) 發送請求時，必須攜帶 `User-Agent` 並關閉 SSL 驗證 (`verify=False`)，以避免憑證阻擋。
    *   向 TAIFEX (期交所) 發送表單請求時，需正確帶入 POST 參數 `queryDate` 與 `queryType`。

## 2. 核心邏輯 (Core Logic) - 籌碼計算系統
### 2.1 口數比例換算 (Proportional Calculation)
必須包含大台、小台與微台，並嚴格遵循標準契約比例轉換為大台當量再進行總和：
*   **總口數公式** = `臺股期貨 (TX)` + `(小型臺指期貨 (MTX) / 4)` + `(微型臺指期貨 (TMF) / 20)`

### 2.2 日/夜盤分離邏輯 (Session Separation)
*   **日盤期貨多空單數據**：期交所「當日買賣超」計算基準包含「前一交易日的夜盤」。因此，為取得純粹的白日盤交易數據，必須使用相減邏輯：
    *   `純日盤數據 = 當日買賣超總和 - 夜盤買賣超總和`
*   **現貨買賣超**：抓取 TWSE `BFI82U` 之「外資及陸資(不含外資自營商)」項目，並強制將其原始金額除以 `100,000,000`，使單位換算為「億」元，並四捨五入至小數點第一位。

## 3. UI 與視覺化規範 (UI & Visualization - LOCKED)
### 3.1 顏色與介面佈局
*   **整體風格**：採用亮白背景 (`#ffffff`) 與純黑字體 (`#000000`) 的高對比設計，卡片邊框設定為不隨系統深淺色變更，確保在任何瀏覽器主題下皆具備極佳閱讀性。
*   **字體設定**：統一使用「微軟正黑體 (`Microsoft JhengHei`)」及 `Arial` 加大字級，提升數字清晰度。
*   **警示高亮 (Highlight)**：核心觀測點「**日盤期貨多空單數據**」那一列，必須以強烈的亮黃色背景 (`#ffff00`) 搭配純黑粗體字獨立強調。
*   **資訊來源超連結**：右下角必須常駐淺灰色的資料來源超連結，包含期交所日/夜盤與證交所頁面，供快速核實。

### 3.2 多空訊號與提醒 (Signals & Alerts)
底部大字體之多空判定，需依據「日盤期貨多空單數據」與「外資現貨當日買賣超」之正負號進行聯合判斷：
1.  **雙多 (皆 > 0)**：顯示 `偏多看待，逢低買進`（紅色粗體，大小 `28px`，帶文字陰影）。
2.  **雙空 (皆 < 0)**：顯示 `偏空看待，逢低買進`（綠色粗體，大小 `28px`，帶文字陰影）。
3.  **多空不一致**：顯示 `多空不明`（橘黃色粗體，大小 `28px`，帶文字陰影）。

### 3.3 轉向加重提醒 (First-Time Change Alert)
*   **定義**：當判斷出的最新訊號與「前一交易日」的訊號不一致（例如由空轉多、由不明轉多等狀態改變）時，視為發生第一次轉向。
*   **實作**：系統於啟動或切換日期時，必須自動往前追溯日期（跳過週末）並計算出前一個有效交易日的訊號。狀態一旦發生改變，必須在畫面底部渲染 Streamlit 原生的警告區塊：`⚠️ 第一次轉向，須加重提醒！`。

## 4. 錯誤處理與防呆機制 (Error Handling)
*   **非交易日攔截**：若使用者於選單選擇了週末或國定假日，導致 TAIFEX 或 TWSE 查無資料，系統應攔截錯誤並顯示：「未能獲取該日完整資料，可能為非交易日。」以防程式崩潰。
*   **網路連線容錯**：所有外部 HTTP 請求皆應設立逾時門檻 (`timeout=10`)，避免網站當機時導致爬蟲卡死。
